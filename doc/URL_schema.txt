PDMP "Spike" URL schema, based on hData

Authentication
==============

HTTP Basic authentication is enabled for API requests.  Web browser
requests use cookie-based authentication.

Patient Query/Match
===================

GET
---
/people - HTML index of people
/people - Atom feed of people
 HTTP header "Accept: application/atom+xml"

Patient Data Retrieval
======================

GET
---
/people/{id} - person's HTML index page
/people/{id} - person's atom index
 HTTP header "Accept: application/atom+xml"
/people/{id}/root.xml - person's hData root document

Each person has 3 sections: c32 (composed of both demographics and
medication in a single document), demographic, and medication.

/people/{id}/c32 - person's c32 section
/people/{id}/c32 - person's c32 section atom feed
 HTTP header "Accept: application/atom+xml"
/people/{id}/c32/c32 - person's composed (i.e. all-in-one) green c32 record
 HTTP header "Accept: application/xml"

/people/{id}/demographic - person's demographic section
/people/{id}/demographic - person's demographic section atom feed
 HTTP header "Accept: application/atom+xml"
/people/{id}/demographic/demographic - person's green demographic record
 HTTP header "Accept: application/xml"

/people/{id}/medication - person's medication section
/people/{id}/medication - person's medication section atom feed
 HTTP header "Accept: application/atom+xml"
/prescriptions/{id} - green medication record for one prescription
 HTTP header "Accept: application/xml"
